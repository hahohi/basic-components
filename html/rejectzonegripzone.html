Reject zone
Touch event 가 발생하지 않는 영역으로 Curved edge screen 및 좁은 Bezel 로 인해 파지 시 터치 오동작이 발생하는 것을 방지하기 위함.
- Default : 하단 70% 영역 좌우 Edge 2.22%(8px) / 상단 30% 영역 및 3rd party 화면은 0.69%(2.5px) 일괄 적용
- Landscape : 상하 Edge 0.69%(2.5px) 영역 긴 변 을 모두 reject zone 으로 적용
Reject zone
- 3×4 key layout 등 먼 버튼을 눌러야 하는 Layout 에서는 파지 및 손바닥에 의해 터치 간섭이 일어나지 않도록 좌우 Reject zone 을 넓게 주고 하단 버튼 배치 시 주의 필요
- 기본 2.22% (8px) Reject zone 에 추가로 최대 3.33% (12px) 이내에서 정의 가능 / 경우에 따라 더 확장된 Reject zone 정의 가능 (Ripple effect 영역 고려하여 설정 필요)
- Reject zone 은 사각 Block 으로 정의
- 한 변에 1 단계까지 설정 가능 (0%+1 단계 가능) 0% 로 설정하더라도 단말 기본값 0.69% (2.5px)가 적용됨
- 비대칭은 기본적으로 미적용 (Edge handler 는 별도 예외로 처리)
Grip zone
Long press event 가 발생하지 않는 영역으로 오동작 방지를 위한 palm rejection, 3 finger rejection 활용 영역
- Default: 4.17%(15px) 일괄 적용 {3rd party: 2.78%(10px)}
- Landscape : 4.17%(15px) 일괄 적용 {3rd party: 2.78%(10px)}
