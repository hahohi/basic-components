URL 확인
화면이 새로고침 되고 다시 항목을 확인하면 저장소 명칭 기준으로 URL이 생성된 것을 확인할 수 있습니다.
우측 화면의 URL 명칭 기준으로 https://hahohi.github.io/basic-components/에 접속한다면,
로컬 저장소 'basic-components' 폴더 내 'index.html'에 접근하는 것과 같습니다.
해당 프로젝트는 'basic-components' 폴더 내 4개의 하위폴더(assets, custom, html, measure)에
내용이 나뉘어 정리되어 있고 'index.html'는 'basic-components' 폴더 내 'html' 폴더에 위치하고 있습니다.
따라서, 올바른 URL은 https://hahohi.github.io/basic-components/ 뒤에 html/index.html 을
추가하여 아래와 같은 URL을 입력합니다.
https://hahohi.github.io/basic-components/html/index.html
GitHub Pages 설정 변경
'Settings' 로 진입한 뒤 나타나는 페이지에서 스크롤을 아래로 내리면 'GitHub Pages' 항목을
확인할 수 있습니다.
'Source > None' 으로 설정되어있는 메뉴를 클릭하여 'master branch'를 선택합니다.
Settings로 이동
웹 사이트로 이동하면 현재 저장소에 업로드 되어있는 파일들을 확인할 수 있습니다.
우측 상단의 'Settings'를 클릭하여 이동합니다.
GitHub 웹으로 이동
GitHub 웹 관리 페이지로 이동하여 세팅을 진행해야 합니다.
'View on GitHub(⌘⇧G)를 클릭하여 웹 사이트로 이동합니다.
웹 호스팅
Commit
동기화
동기화 폴더에 파일을 삽입 후 Changes 탭을 보면 추가된 파일들을 확인할 수 있습니다.
그 후 하단의 Commit 영역을 작성해야 합니다. 아래와 같이 작성합니다.
작성을 마치고 'Commit to Master' 를 클릭하여 완료합니다.
Summary (required) : 프로젝트 인원들이 식별할 수 있는 명칭. 업로드 날짜 혹은 버젼명
Description : 세부 내용 작성
최종 발행 (Publish)
Commit 을 진행하면 Changes 탭 목록의 리스트들이 사라졌지만 아직 발행된 것이 아니며,
발행 전 대기열에 올라가있는 상태입니다.
상단 중앙의 'Publish repository' 버튼을 클릭하면 저장소에 파일이 업로드 됩니다.
Publish를 클릭하면 처음 한번만 저장소의 공개/비공개 선택 여부를 묻는 창이 나타납니다.
만약 무료로 저장소를 사용할 경우 'Keep this code private'는 체크를 해제합니다.
파일 삽입
Finder 또는 탐색기를 통해 프로젝트 파일(assets, custom, html, measure)을 복사하여
Local Path 내 생성된 저장소 폴더로 이동시킵니다.
편의상 프로젝트 파일이 있는 임의의 폴더를 'Project-Folder', 동기화 되는 폴더는
'basic-components' 라고 하였습니다.
저장소 (Repository) 기능
비어있는 저장소가 생성되었습니다.
앞 단계에서 지정한 Local Path 내 폴더에 파일을 추가하면 Changes 탭에서는 기존 파일과 비교하여
변경이 된 파일들을 알려줍니다.
History는 변경된 파일의 목록과 내용이 시간 별로 기록되어 오류 체크 시 용이하게 사용할 수 있습니다.
저장소 (Repository) 생성
앱의 메인 화면에서 'Create a New Repository on your Hard Drive...' (⌘N)를 클릭하면
'Create a New Repository' 창이 나타납니다. 아래와 같이 작성합니다.
작성을 마치고 'Create Repository'를 클릭하여 생성을 완료합니다.
Name : 임의의 프로젝트 명칭 (해당 명칭으로 지정한 Local Path 안에 폴더가 생성됩니다. 띄어쓰기 금지, 소문자 권장)
Description : 임의의 프로젝트에 대한 설명
Local Path : 웹 저장소에 업로드할 파일이 동기화될 임의의 로컬 위치 지정
Initialize this repository with a README : 체크함
Git Ignore : None
License : None
로그인
설치 후 실행하여 상단 메뉴 막대에서 Github Desktop > Preference...(⌘,) 를 클릭합니다.
Preferences 창에서 Accounts 탭으로 접근하여 로그인합니다.
만약 계정이 없다면 생성한 뒤에 로그인합니다.

설치
대표적인 Git 저장소로, 무료 호스팅 기능을 제공합니다.
무료로 제공하는 저장소 기능을 이용하여 웹 문서를 업로드하고 주소를 부여받을 수 있습니다.
저장소를 생성하고 관리하는 방법은 여러가지가 있습니다. 다만 웹에서 지원하는 기능은 제한적이고
터미널을 통해 관리하는 것은 복잡하므로 해당 프로젝트에서는 'Github Desktop'이라는 앱을 이용하여 진행합니다.
